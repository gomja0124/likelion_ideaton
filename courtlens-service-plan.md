## 서비스 기획 배경

한국 프로스포츠는 오랫동안 엘리트 스포츠 구조 안에서 성장해왔다. 팬들은 경기를 좋아하고 선수에게 관심을 가지지만, 실제로는 선수, 감독, 코치, 구단이 경험하는 스포츠의 세계와 팬들이 경기장에서 느끼는 세계 사이에 거리가 있다.

특히 농구는 경기 속도가 빠르고 전술, 선수 역할, 흐름 변화가 중요하다. 처음 보는 팬은 누가 잘했는지, 왜 그 장면이 중요한지, 어떤 선수를 중심으로 경기를 봐야 하는지 이해하기 어렵다. 반대로 구단은 팬들이 어떤 장면에 반응하고, 어떤 선수에게 입덕하며, 어떤 불편이나 니즈를 느끼는지 SNS와 커뮤니티를 일일이 확인하지 않으면 파악하기 어렵다.

CourtLens는 이 간극을 줄이기 위한 서비스다. 팬에게는 경기 전후로 이해하기 쉬운 하이라이트와 입덕 콘텐츠를 제공하고, 구단에게는 팬 반응과 경기 장면 데이터를 기반으로 이벤트, 굿즈, 스폰서십, 팬 경험 개선 전략을 제안한다.

## 기획 목적

CourtLens의 목적은 “AI로 스포츠 진입장벽을 낮추면, 팬은 경기를 더 재미있게 보고 구단은 팬을 더 오래 볼 수 있다”는 가설을 서비스로 검증하는 것이다.

이 서비스는 단순히 하이라이트 영상을 모아 보여주는 앱이 아니다. 공식 경기 기록, play-by-play 이벤트, 선수 정보, 팬 반응을 연결해 팬이 이해할 수 있는 언어로 경기의 맥락을 번역한다. 동시에 구단이 팬의 반응을 수익화와 팬 경험 개선으로 연결할 수 있도록 돕는다.

## 기대 효과

### 팬 관점

- 농구를 잘 몰라도 경기에서 누가 잘했는지 이해할 수 있다.
- 경기 전에는 직관메이트를 통해 혼자 직관하는 부담을 줄일 수 있다.
- 경기 후에는 AI 하이라이트와 입덕패스를 통해 선수, 전술, 경기 흐름을 쉽게 이해할 수 있다.
- 댓글과 반응을 통해 다른 팬들과 같은 장면을 중심으로 대화할 수 있다.

### 구단 관점

- 팬 반응을 SNS와 커뮤니티에서 수작업으로 찾지 않아도 된다.
- 어떤 선수, 장면, 경기 경험이 팬의 관심을 만드는지 확인할 수 있다.
- 시즌 중 빠르게 실행할 수 있는 단기 이벤트를 기획할 수 있다.
- 라운드 종료 이벤트, 프랜차이즈 스타 마케팅, 굿즈 테스트, 스폰서 패키지의 근거 데이터를 확보할 수 있다.

### 서비스 관점

- 팬 앱은 무료 또는 저가 진입으로 사용자 기반을 만든다.
- 주요 수익은 구단 대상 B2B SaaS와 스폰서 패키지에서 발생한다.
- 팬 경험 데이터가 쌓일수록 구단에게 제공하는 분석 가치가 커진다.

## 주요 기능

### 1. 하이라이트

경기 후에 열리는 핵심 기능이다. KBL 경기 기록과 play-by-play 데이터를 기반으로 중요한 장면을 찾고, 해당 장면에 영상 클립을 연결한다.

- 경기 선택
- 선수 선택
- play-by-play 기반 주요 장면 확인
- 이 경기에서 잘한 선수 추천
- 하이라이트 원본 생성
- 생성된 하이라이트를 입덕패스, 구단 이벤트, 다음 경기 직관메이트 소재로 복제 배포

모델 개발은 별도 범위로 둔다. 서비스 기획 단계에서는 `공식 경기 기록을 기준으로 중요한 장면을 찾고, 해당 시간대 영상 클립을 연결한다`는 흐름을 중심으로 설계한다.

### 2. 입덕패스

경기를 잘 모르는 팬을 위한 쉬운 해설 기능이다. 하이라이트 장면을 초보자 언어로 풀어주고, 팬 댓글을 통해 Fan Voice를 수집한다.

- 오늘 처음 봐도 이해되는 경기 요약
- 선수별 입덕 포인트 설명
- 장면별 쉬운 해설
- 팬 댓글 작성
- 댓글 기반 Fan Voice 수집

### 3. 직관메이트

경기 전에 열리는 팬 매칭 기능이다. 처음 직관하는 팬, 혼자 직관하기 부담스러운 팬, 같은 선수를 좋아하는 팬을 연결한다.

- 직관 날짜 선택
- 좌석 위치 선택
- 관심 선수 선택
- 모집글 작성
- AI 모집글 초안 생성
- 같은 선수/장면에 반응한 팬에게 추천

### 4. 구단 대시보드

구단이 비용을 지불하는 B2B 기능이다. 팬 앱에서 발생한 반응과 경기 데이터를 분석해 구단의 이벤트, 굿즈, 스폰서십, 팬 경험 개선 전략으로 연결한다.

- Fan Insight: 선수별 반응, 저장, 공유, 댓글, 입덕 흐름 분석
- Fan Voice: 팬 댓글과 불편 사항을 주제별로 정리
- Revenue Playbook: 단기 이벤트, 라운드 이벤트, 장기 프랜차이즈 마케팅 추천
- Event Builder: 구단이 바로 실행할 수 있는 단기 이벤트 초안 생성
- Sponsor Package: 인기 장면과 선수 반응을 기반으로 스폰서 제안 구성
- Performance: CTA 클릭, 이벤트 참여, 굿즈 반응, 예상 매출 기여 추적

## 주요 고객(타겟층)

### 1차 사용자: 농구 팬

- 농구를 이제 보기 시작한 초보 팬
- 특정 선수에게 관심이 생긴 신규 팬
- 직관은 가고 싶지만 혼자 가기 부담스러운 팬
- 하이라이트만 보다가 실제 경기로 넘어가고 싶은 팬

### 주요 고객: 프로스포츠 구단

- 팬 경험을 개선하고 싶은 KBL 구단
- 시즌 중 단기 이벤트를 빠르게 기획해야 하는 구단 마케팅팀
- 선수별 팬 반응을 바탕으로 굿즈나 콘텐츠를 만들고 싶은 구단
- 스폰서십 제안에 팬 반응 데이터를 활용하고 싶은 구단

### 확장 고객: 스폰서/브랜드

- 특정 선수나 경기 장면에 반응한 팬에게 브랜드를 노출하고 싶은 스폰서
- 구단 이벤트와 연계된 참여형 캠페인을 원하는 브랜드
- 스포츠 팬덤 기반 타겟 마케팅을 원하는 기업

## 서비스 흐름

### 경기 전

1. 팬이 다음 경기 정보를 확인한다.
2. 관심 선수와 좌석 성향을 선택한다.
3. 직관메이트 모집글을 작성하거나 추천받는다.
4. 구단은 직관메이트 수요와 좌석 선호를 확인한다.

### 경기 후

1. KBL 공식 경기 기록과 play-by-play 로그를 기반으로 주요 장면을 추출한다.
2. 하이라이트 메이커가 중요한 선수와 장면을 추천한다.
3. 팬은 AI 하이라이트와 입덕패스를 본다.
4. 팬 댓글과 저장/공유 반응이 Fan Voice로 쌓인다.
5. 구단은 해당 반응을 단기 이벤트, 굿즈, 스폰서 패키지로 전환한다.

- Fan Voice 리포트

- 티켓 CTA 클릭
- 굿즈 테스트 클릭
- 이벤트 참여
- 스폰서 이벤트 참여

### 스폰서 패키지

인기 장면, 선수 반응, 팬 참여 데이터를 기반으로 스폰서 상품을 구성한다.

- 오늘의 하이라이트 스폰서
- 라운드 MVP 팬 투표 스폰서
- 프랜차이즈 스타 굿즈 스폰서
- 직관메이트 매칭 이벤트 스폰서

## 모델 개발 범위

모델 개발은 서비스 기획과 분리한다. 아이디어톤 발표에서는 모델을 직접 완성하는 것보다, 어떤 데이터를 어떤 방식으로 연결해 서비스 가치로 바꾸는지가 중요하다.

별도 개발할 모델의 방향은 다음과 같다.

- KBL play-by-play 이벤트 구조화
- 경기 시간과 영상 시간대 매칭
- 장면별 중요도 점수 계산
- 선수별 영향도 계산
- 하이라이트 후보 추천
- 초보자용 설명 생성

서비스 기획서에서는 이 모델을 `Play-by-Play Highlight Linker`로 정의한다.

## MVP 기획

아이디어톤 MVP의 목표는 완전한 영상 분석 모델을 만드는 것이 아니다. CourtLens가 어떤 사용자 문제를 해결하고, 팬 앱과 구단 대시보드가 어떻게 연결되며, 이 연결이 왜 수익화 가능성이 있는지 보여주는 것이다.

MVP는 `팬 경험 프로토타입 + 구단 대시보드 프로토타입 + play-by-play 기반 하이라이트 연결 구조`로 정의한다.

## MVP 핵심 가설

1. 농구 초보 팬은 경기의 모든 정보를 원하는 것이 아니라, “오늘 누구를 보면 되는지”와 “왜 이 장면이 중요한지”를 먼저 알고 싶어한다.
2. 경기 후 하이라이트는 단순 소비 콘텐츠가 아니라 다음 경기 직관, 입덕 콘텐츠, 구단 이벤트의 출발점이 될 수 있다.
3. 구단은 팬 반응을 모아 단기 이벤트, 굿즈 테스트, 스폰서 패키지로 전환할 수 있다면 B2B SaaS 비용을 지불할 이유가 있다.

## MVP 사용자 흐름

### 경기 전: 직관메이트

1. 팬이 다음 경기 정보를 본다.
2. 관심 선수, 직관 날짜, 좌석 위치, 응원 성향을 선택한다.
3. AI가 모집글 초안을 생성한다.
4. 같은 선수나 같은 장면에 반응한 팬에게 모집글이 추천된다.
5. 구단은 경기 전 좌석 선호, 동행 수요, 초보 팬 니즈를 확인한다.

### 경기 후: 하이라이트

1. 팬이 종료된 경기 중 하나를 선택한다.
2. 서비스가 공식 경기 기록과 play-by-play 이벤트를 기준으로 주요 장면 후보를 보여준다.
3. 팬은 관심 선수를 선택한다.
4. AI가 해당 선수 중심의 하이라이트 원본을 생성한다.
5. 생성된 원본은 입덕패스, 구단 이벤트, 다음 경기 직관메이트 소재로 재활용된다.

### 경기 후: 입덕패스

1. 팬은 하이라이트 장면을 초보자용 설명으로 본다.
2. “이 선수가 왜 잘했는지”, “이 장면이 왜 중요한지”를 짧게 이해한다.
3. 팬은 댓글로 입덕 포인트나 궁금한 점을 남긴다.
4. 댓글과 저장/공유 반응은 Fan Voice로 수집된다.

### 구단 운영

1. 구단은 선수별 반응, 하이라이트 저장, 댓글, 직관메이트 클릭을 확인한다.
2. 서비스는 단기 이벤트, 라운드 이벤트, 장기 프랜차이즈 마케팅 후보를 추천한다.
3. 구단은 추천 액션을 바탕으로 이벤트 초안이나 스폰서 제안서를 만든다.
4. CTA 클릭, 이벤트 참여, 굿즈 반응을 통해 성과를 확인한다.

## MVP 화면 구성

### 팬 앱

- 하이라이트: 경기 후 열리는 핵심 화면. 경기 선택, 선수 선택, play-by-play 주요 장면, 하이라이트 생성 버튼을 포함한다.
- 입덕패스: 하이라이트를 초보자 언어로 번역하는 화면. 댓글 기능을 통해 Fan Voice를 수집한다.
- 직관메이트: 경기 전 열리는 화면. 날짜, 좌석, 관심 선수, 모집글 작성을 포함한다.

### 구단 대시보드

- Fan Insight: 어떤 선수와 장면이 팬 반응을 만들었는지 보여준다.
- Fan Voice: 댓글과 불편 사항을 주제별로 정리한다.
- Revenue Playbook: 단기 이벤트, 라운드 이벤트, 장기 마케팅 후보를 제안한다.
- Performance: CTA 클릭, 이벤트 참여, 굿즈 반응, 예상 매출 기여를 보여준다.

## MVP 데이터 구조

MVP에서는 실제 KBL API 연동을 완성하지 않고, KBL 경기 기록 페이지 형태를 참고한 샘플 데이터를 사용한다.

### 경기 데이터

- 경기 ID
- 경기 날짜
- 홈팀/원정팀
- 최종 스코어
- 쿼터별 스코어
- 경기 상태: 경기 전, 진행 중, 경기 후

### play-by-play 이벤트 데이터

- 쿼터
- 경기 시계 시간
- 선수명
- 액션 종류: 득점, 어시스트, 리바운드, 스틸, 블록, 교체
- 관련 선수: 어시스트, 리바운드, 파울 유도 등
- 점수 차
- 하이라이트 후보 점수
- 연결 영상 정보: 영상 URL, 시작 초, 종료 초

### 팬 반응 데이터

- 저장
- 공유
- 댓글
- 선수 클릭
- 직관메이트 클릭
- 티켓/굿즈 CTA 클릭

## MVP에서 보여줄 핵심 데모

### 데모 1: 농구를 잘 모르는 팬이 경기 후 하이라이트를 이해하는 흐름

1. 하이라이트 탭에서 종료된 경기를 선택한다.
2. 서비스가 “이 경기 입덕 포인트 선수”를 보여준다.
3. play-by-play 기반 핵심 장면을 보여준다.
4. 팬이 선수를 선택하고 하이라이트를 생성한다.
5. 입덕패스에서 해당 장면이 쉬운 언어로 설명된다.

### 데모 2: 경기 전 직관메이트 흐름

1. 직관메이트 탭으로 이동한다.
2. 다음 경기, 좌석, 관심 선수, 성향을 선택한다.
3. 모집글 초안을 생성한다.
4. 같은 선수/장면에 반응한 팬에게 추천된다는 구조를 보여준다.

### 데모 3: 구단이 팬 반응을 수익화로 연결하는 흐름

1. 구단 대시보드에서 선수별 반응을 확인한다.
2. Revenue Playbook에서 단기 이벤트 후보를 본다.
3. Event Builder로 이벤트 초안을 만든다.
4. Performance에서 CTA/참여/예상 매출 기여를 확인한다.

## MVP 성공 기준

### 팬 앱 기준

- 사용자가 하이라이트, 입덕패스, 직관메이트의 역할을 구분할 수 있어야 한다.
- 경기 전 기능과 경기 후 기능의 차이가 명확해야 한다.
- 초보 팬이 “이 선수/장면을 보면 되겠다”는 판단을 할 수 있어야 한다.
- 하이라이트가 다음 액션인 입덕패스, 직관메이트, 이벤트로 연결되어야 한다.

### 구단 대시보드 기준

- 구단이 팬 반응을 한눈에 볼 수 있어야 한다.
- 추천 액션이 단순 아이디어가 아니라 실행 단위로 보여야 한다.
- 단기 이벤트, 라운드 이벤트, 장기 마케팅이 구분되어야 한다.
- 수익화 가능성이 티켓, 굿즈, 스폰서, 이벤트 참여로 설명되어야 한다.

## MVP에서 제외할 것

- 실제 영상 AI 분석 모델 완성
- 실제 KBL API 자동 수집
- 실제 결제/예매/굿즈몰 연동
- 실시간 경기 중 하이라이트 자동 생성
- 모든 종목 지원
- 구단 관리자 권한/계정 시스템

위 항목은 서비스 확장 단계에서 다룬다. 아이디어톤 MVP에서는 사용 흐름과 사업성을 보여주는 데 집중한다.

## MVP 이후 확장 로드맵

### 1단계: KBL 경기 기록 기반 MVP

- KBL 경기 기록 페이지 구조를 참고한 샘플 play-by-play 데이터 사용
- 수동 영상 링크 연결
- 팬 앱 3개 탭과 구단 대시보드 프로토타입 완성

### 2단계: Play-by-Play Highlight Linker 개발

- KBL play-by-play 이벤트 자동 구조화
- 경기 시계 시간과 영상 시간대 매칭
- 장면별 중요도 점수 계산
- 선수별 영향도 계산

### 3단계: 구단 파일럿

- 특정 구단 또는 특정 경기 데이터로 테스트
- 팬 댓글과 하이라이트 반응 수집
- 단기 이벤트 추천과 CTA 성과 추적

### 4단계: 종목 확장

- 농구에서 축구, 배구, 야구로 확장
- 종목별 play-by-play 구조에 맞는 하이라이트 기준 추가
- 구단별 팬덤 데이터 비교 분석

## 한 문장 정리

CourtLens는 공식 경기 기록과 팬 반응을 연결해, 팬에게는 쉬운 경기 이해를 제공하고 구단에게는 팬 경험 기반 수익화 전략을 제공하는 스포츠 팬덤 플랫폼이다.
