# CourtLens IR Draft

## 1. Core Concept

프로젝트명:

코트렌즈(CourtLens)

주제 문장:

> AI로 스포츠 진입장벽을 낮추면, 팬은 경기를 더 재밌게 보고 구단은 팬을 더 오래 볼 수 있다.

한 줄 소개:

> CourtLens는 KBL 팬이 좋아하는 팀과 선수를 고르면 AI가 개인화 하이라이트와 쉬운 전술 해설을 생성하고, 그 관심을 직관메이트와 구단 수익화로 연결하는 농구 팬 경험 플랫폼입니다.

핵심 포지셔닝:

CourtLens의 1차 사용자는 팬이다. 팬이 앱에서 충분히 유용함을 느껴야 행동 데이터가 쌓이고, 구단이 비용을 지불할 이유도 생긴다. 따라서 팬 앱은 단순한 경기 기록 앱이 아니라, 라이트 팬이 농구를 더 쉽게 이해하고 좋아하는 선수 중심으로 경기를 소비하도록 돕는 제품이어야 한다.

수익 모델은 팬 유료 구독보다 B2B 중심으로 설계한다. 팬에게는 무료 또는 낮은 장벽의 개인화 경험을 제공하고, 구단에게는 팬의 관심 선수, 하이라이트 반응, 직관 의도, 굿즈/티켓 전환 가능성을 분석하는 대시보드를 제공한다.

핵심 고객 구조:

- 1차 사용자: KBL을 더 쉽게 보고 싶은 라이트 팬, 특정 팀/선수 중심으로 경기를 소비하는 팬
- 지불 고객: 팬을 더 오래 붙잡고 티켓·굿즈·스폰서 매출을 키우고 싶은 구단
- 확장 고객: 구단 팬덤의 몰입 장면과 팬 반응을 활용하고 싶은 스폰서 브랜드

핵심 메시지:

> AI가 좋아하는 선수의 장면을 이해 가능한 콘텐츠로 만들면 팬은 경기를 더 재밌게 보고, 구단은 그 관심을 다음 직관과 수익화로 연결한다.

## 2. Current Prototype State

현재 구현된 프로토타입은 `팬 앱 + 구단 대시보드` 구조다. 팬 앱은 실제 KBL 앱과 유사한 기본 흐름을 가지되, CourtLens가 집중하는 기능인 `하이라이트 분석`과 `직관메이트`를 첫 화면에서 보여준다.

현재 구현 완료:

- KBL 공식 API 기반 데이터 최신화
  - 전체 선수 마스터 데이터
  - 현재 등록 선수 데이터
  - 2025-2026 정규시즌 선수 기록 데이터
  - 창원 LG와 수원 KT 선수/기록 필터링
- 마이페이지
  - 로그인된 사용자라고 가정
  - 선호팀을 `창원 LG 세이커스` 또는 `수원 KT 소닉붐`으로 변경
  - 선택 팀은 브라우저 저장소에 저장되어 새로고침 후에도 유지
- 팬 앱 홈
  - 팀 컬러 자동 변경
  - 팀별 주요 선수/기록 카드 자동 반영
  - 팀 로스터 기반 선수 칩 렌더링
- AI 하이라이트 생성
  - 사용자가 여러 선수를 선택
  - `하이라이트 생성` 클릭
  - AI가 영상을 만든다는 전제로 `방금 생성된 클립` 영역에 새 카드 추가
  - 생성된 클립을 누르면 작전판AI 분석 패널이 해당 클립 설명으로 갱신
- 작전판AI
  - 초보 모드/농잘알 모드 전환
  - 선택 장면의 쉬운 해설과 전술적 설명 제공
- 입덕패스
  - 선택 팀의 핵심 선수와 관전 포인트를 초보자용으로 요약
- 직관메이트
  - `같이 직관가실 분 구해요!` 모집 흐름
  - 경기/좌석/응원 성향/관심 하이라이트 기반 메이트 추천 컨셉
- 구단 대시보드
  - 등록 선수, 기록 선수, AI 클립 후보 수
  - 선수별 팬 반응
  - 콘텐츠 자동 생성 큐
  - 장면별 수익화 추천

현재 프로토타입에서 아직 개발하지 않은 기능:

- 실제 영상 업로드 및 자동 클립 추출
- 실제 play-by-play와 영상 타임라인 동기화
- Fan Voice 의견함/투표 화면
- 실제 예매/굿즈몰 연동
- 실제 구단 관리자 캠페인 승인 플로우
- 사용자 계정/서버 저장소 기반 선호팀 관리

따라서 IR에서는 `현재 시연 가능한 MVP`와 `향후 구현할 AI/데이터 파이프라인`을 명확히 구분해야 한다.

## 3. Problem

프로농구는 재미없는 스포츠가 아니다. 문제는 팬이 재미를 느끼기까지 넘어야 하는 장벽이 높고, 구단은 팬의 관심을 실제 행동과 매출로 연결하기 어렵다는 점이다.

팬의 문제:

- 전체 경기를 볼 시간은 부족하다.
- 좋아하는 선수의 장면만 빠르게 모아보기 어렵다.
- 픽앤롤, 스페이싱, 미스매치 같은 전술을 모르면 좋은 플레이를 놓친다.
- 경기 결과는 알 수 있지만, "왜 이 장면이 좋았는지"까지 알려주는 서비스는 부족하다.
- 혼자 직관을 가기 부담스럽고, 나와 비슷한 관심사의 팬을 찾기 어렵다.
- 기존 스포츠 앱은 일정, 기록, 뉴스 중심이라 라이트 팬이 앱을 켤 명확한 이유가 약하다.

구단의 문제:

- 팬들이 어떤 선수, 장면, 콘텐츠에 반응하는지 실시간으로 알기 어렵다.
- 선수별 팬 관심이 티켓, 굿즈, 이벤트로 이어지는 흐름이 약하다.
- 라이트 팬을 코어 팬으로 전환시키는 설명형 콘텐츠가 부족하다.
- 팬의 직관 의도와 동행 니즈를 파악하기 어렵다.
- 스폰서에게 "우리 팬들이 무엇에 몰입하는지"를 데이터로 설득하기 어렵다.

문제 정의:

> 팬에게는 좋아하는 선수 중심으로 경기를 쉽게 이해하고 다음 행동까지 이어가는 경험이 필요하고, 구단에게는 그 팬 관심을 오래 유지하며 수익화할 도구가 필요하다.

## 4. Key Insight

CourtLens의 방향성은 "팬을 위한 경험"과 "구단을 위한 수익화"를 분리하지 않는 것이다.

핵심 인사이트:

1. 팬은 기록표보다 "내가 좋아하는 선수의 장면과 그 의미"에 반응한다.
2. 라이트 팬은 장면을 이해할수록 선수를 더 기억하고, 경기를 더 자주 본다.
3. 좋아하는 선수의 하이라이트를 본 직후가 티켓/굿즈/직관메이트로 연결하기 가장 좋은 순간이다.
4. 혼자 직관이 부담스러운 팬에게는 정보보다 함께 갈 사람과 명확한 관전 포인트가 더 중요할 수 있다.
5. 구단은 단순 조회수보다 "어떤 팬이 어떤 선수와 장면에 반응했고, 다음에 무엇을 할 가능성이 있는지"가 필요하다.

CourtLens가 만드는 선순환:

> 팬이 팀과 선수를 고른다 → AI가 개인화 하이라이트와 쉬운 해설을 만든다 → 팬이 선수를 더 좋아하게 된다 → 직관메이트/예매/굿즈로 이어진다 → 구단은 팬 관심과 전환 데이터를 얻는다 → 더 정교한 콘텐츠와 캠페인을 만든다

## 5. Product Structure

CourtLens는 팬 앱과 구단 대시보드로 구성된다. 팬 앱은 팬이 실제로 쓰는 프론트 스테이지이고, 구단 대시보드는 팬 행동을 분석해 수익화 전략으로 바꾸는 백오피스다.

### A. 팬 앱

팬 앱의 목적은 팬에게 명확한 결과를 주는 것이다. 팬은 앱을 켰을 때 "내 팀에서 오늘 누구를 보면 되는지", "내가 고른 선수의 장면이 무엇인지", "방금 플레이가 왜 좋은지", "다음 직관을 누구와 갈 수 있는지"를 바로 알아야 한다.

현재 프로토타입 기준 핵심 화면:

1. 마이페이지
   - 로그인된 사용자라고 가정
   - 선호팀 변경: 창원 LG, 수원 KT
   - 팀 변경 시 색상, 선수 목록, 기록 리더, 직관메이트 문구 자동 변경

2. 내 팀/선수 선택
   - 현재 팀 로스터에서 다양한 선수 칩 제공
   - 여러 선수를 선택 가능
   - 선택한 선수 조합을 기준으로 AI 하이라이트 생성

3. 내 하이라이트
   - 선택 선수 기반 AI 생성 클립
   - 기존 추천 클립과 새로 생성된 클립을 같은 피드에 표시
   - 예시:
     - "유기상 · 양준석 중심 AI 하이라이트"
     - "아셈 마레이 세컨드 찬스"
     - "데릭 윌리엄스 · 강성욱 중심 AI 하이라이트"

4. 작전판AI
   - 하이라이트 장면을 누르면 AI가 플레이의 의미를 설명
   - 초보 모드: 쉬운 말로 장면 설명
   - 농잘알 모드: 픽앤롤, 킥아웃, 세컨드 찬스, 트랜지션 등 전술 맥락 설명

5. 입덕패스
   - 처음 보는 팬도 따라갈 수 있도록 선수 서사, 주요 장면, 관전 포인트를 짧게 제공
   - "오늘 이 선수를 보면 되는 이유"를 만든다.

6. 직관메이트
   - 하이라이트를 본 뒤 다음 홈경기에서 같이 볼 사람을 찾는 기능
   - 단순 커뮤니티가 아니라 관심 선수, 응원 성향, 좌석 선호, 경기 일정 기반 매칭
   - 예시:
     - "같이 직관가실 분 구해요!"
     - "유기상 3점 보러 가실 분"
     - "아셈 마레이 골밑 싸움 같이 볼 사람"

### B. 구단 대시보드

구단 대시보드는 팬 앱에서 발생한 행동을 분석해 구단이 실행할 수 있는 전략으로 바꾼다.

현재 프로토타입 기준 핵심 화면:

1. 팀 데이터 요약
   - 등록 선수 수
   - 기록 선수 수
   - AI 클립 후보 수
   - 데이터 기준 시즌

2. 선수별 팬 반응
   - 하이라이트 조회/저장/공유를 통합한 반응 점수 목업
   - 팀 변경 시 선수명 자동 변경

3. 콘텐츠 자동 생성 큐
   - 선택 팀의 주요 선수 기반 숏폼/카드뉴스/직관메이트 모집 콘텐츠 제안

4. 장면별 수익화 추천
   - 특정 선수 장면을 직관메이트, 응원석 추천, 입덕패스 시리즈, 굿즈 캠페인으로 연결

향후 발전 기능:

- Fan Voice AI: 팬 의견 자동 분류, 요약, 감성 분석
- Fan Behavior Insight: 선수별 하이라이트 조회율, 생성률, 저장률, 직관메이트 클릭률 분석
- Revenue Playbook: 구단이 바로 실행 가능한 티켓/굿즈/스폰서 액션 추천
- Campaign Builder: 구단 담당자가 추천 액션을 승인하면 팬 앱에 캠페인 노출
- Sponsor Insight: 인기 장면과 팬 반응 기반 스폰서 리포트 생성

## 6. Prototype Correction Points

현재 개발 사항을 기준으로 IR과 UI에서 수정해야 할 부분은 다음과 같다.

### 1. Fan Voice는 MVP 시연 범위에서 후순위로 내려야 한다

기존 문서는 Fan Voice를 핵심 기능으로 강하게 설명하고 있었지만, 현재 구현된 프로토타입은 `개인화 하이라이트`, `작전판AI`, `직관메이트`, `구단 대시보드`에 초점이 맞춰져 있다. 발표에서는 Fan Voice를 "확장 기능" 또는 "구단 대시보드 고도화 단계"로 두는 편이 더 설득력 있다.

수정 방향:

- 100초 발표에서는 Fan Voice보다 `AI 하이라이트 생성 → 쉬운 해설 → 직관메이트` 흐름을 강조
- Fan Voice는 구단 수익화/운영 개선을 위한 2차 기능으로 언급

### 2. "KBL 한 팀 기준"이 아니라 "두 팀 전환 가능한 MVP"로 바꿔야 한다

현재 프로토타입은 창원 LG와 수원 KT를 전환할 수 있다. 따라서 `KBL 한 팀에서 시작`이라는 표현은 현재 구현보다 약하다.

수정 방향:

- MVP 범위: KBL 2개 팀 기준 데이터 검증
- 창원 LG: 기본 로그인 선호팀
- 수원 KT: 마이페이지에서 전환 가능한 두 번째 팀

### 3. 기존 예시 선수명을 현재 데이터와 맞춰야 한다

문서에 남아 있던 허훈, 워니 중심 예시는 현재 UI/데이터와 맞지 않는다.

수정 방향:

- 창원 LG 예시: 유기상, 양준석, 아셈 마레이, 칼 타마요
- 수원 KT 예시: 데릭 윌리엄스, 강성욱, 박준영, 문정현

### 4. 직관메이트는 "CTA"가 아니라 독립 기능으로 키워야 한다

초안에서는 직관메이트가 예매/굿즈 CTA에 가깝게 설명되어 있었다. 현재 구현은 "같이 직관 갈 사람을 찾는 기능"까지 확장됐다.

수정 방향:

- 직관메이트를 핵심 차별 기능으로 승격
- 하이라이트 기반 관심사 매칭으로 설명
- 혼직관 부담, 초보 팬 동행, 응원석 취향을 문제로 연결

### 5. AI 하이라이트 생성의 MVP 전제를 명확히 해야 한다

현재 UI는 실제 영상 처리 결과가 아니라 "AI가 생성한다는 전제"의 인터랙션 목업이다.

수정 방향:

- 현재 시연: 선수 선택 기반 AI 생성 카드와 해설 UI
- 실제 구현: 영상/기록/play-by-play/OCR/LLM을 결합한 클립 생성 파이프라인
- 발표에서 목업과 실제 기술 계획을 분리해 설명

### 6. 추가/수정해야 할 인터랙션 요구사항

현재 IR과 프로토타입은 "보여주는 대시보드"에서 "눌러서 실제 액션이 이어지는 대시보드"로 발전해야 한다. 특히 B2B 수익화 설득력을 높이려면 구단 대시보드의 각 카드가 단순 지표가 아니라 실행 가능한 모달, 콘텐츠, 캠페인으로 연결되어야 한다.

팬 앱 수정 요구사항:

1. 직관메이트 모집글 작성
   - `메이트 찾기` 또는 `모집글 쓰기` 버튼 클릭 시 작성 모달을 띄운다.
   - 입력 항목: 직관 날짜, 경기 선택, 좌석 위치, 응원 성향, 관심 선수, 자유 텍스트
   - 작성 완료 시 모집글 카드가 직관메이트 영역에 생성된다.
   - "모집글 초안 생성됨"은 버튼이 아니라 2~3초 후 사라지는 토스트/스낵바로 처리한다.

2. 직관메이트 모집글 카드
   - 작성자의 직관 날짜, 좌석 위치, 관심 선수, 응원 성향을 보여준다.
   - 같은 선수 하이라이트를 본 팬에게 추천될 수 있다는 맥락을 표시한다.
   - IR에서는 혼직관 부담을 낮추는 기능이자 구단 입장에서는 직관 의도 데이터를 만드는 기능으로 설명한다.

구단 대시보드 수정 요구사항:

3. 선수별 팬 반응 상세 모달
   - 선수 카드를 클릭하면 선수 정보가 뜬다.
   - 포함 정보: 선수 기본 정보, 시즌 기록, 팬 반응 점수, 인기 하이라이트, 저장/공유/직관메이트 클릭, 추천 캠페인
   - 이 모달은 구단이 "어떤 선수를 콘텐츠와 굿즈로 밀어야 하는지" 판단하는 화면이다.

4. 등록 선수/기록 선수/클립 후보 상세 모달
   - 상단 요약 카드도 클릭 가능해야 한다.
   - 등록 선수: 현재 팀 로스터 목록, 포지션, 등번호, 주요 선수 여부
   - 기록 선수: 기록 데이터가 있는 선수 목록, 득점/리바운드/어시스트 등 주요 지표
   - 클립 후보: AI 하이라이트 후보 장면 목록, 관련 선수, 예상 콘텐츠 유형, 생성 가능 상태

5. 콘텐츠 자동 생성 큐 실행
   - 큐 항목 클릭 시 실제 생성 콘텐츠 미리보기가 떠야 한다.
   - 예시 결과: 숏폼 제목, 썸네일 문구, 카드뉴스 카피, 선수 소개 문구, 입덕패스 문장, 직관메이트 모집 문안
   - `콘텐츠 생성`, `수정`, `팬 앱에 노출`, `스폰서 패키지로 저장` 같은 후속 버튼을 둔다.

6. 장면별 수익화 추천 실행 모델
   - 추천 액션은 단순 텍스트가 아니라 실행 플로우로 이어져야 한다.
   - 모델: 추천 장면 선택 → 추천 액션 확인 → 캠페인 템플릿 생성 → 구단 담당자 승인 → 팬 앱 노출 → 클릭/전환 추적
   - 예시: "유기상 3점 하이라이트 반응 높음" → "응원석 예매 CTA 생성" → "팬 앱 하이라이트 하단 노출" → "예매 클릭률 측정"

7. 전환 퍼널 상세 정보
   - 전환 퍼널의 각 메뉴/단계를 클릭하면 세부 정보가 떠야 한다.
   - 단계 예시: 하이라이트 생성, 클립 조회, 작전판AI 열람, 직관메이트 클릭, 티켓 CTA 클릭, 굿즈 CTA 클릭
   - 세부 정보: 사용자 수, 전환율, 이탈률, 관련 선수, 관련 콘텐츠, 다음 추천 액션

8. 구단 대시보드 추가 추천 메뉴
   - Campaign Builder: Revenue Playbook 추천을 실제 캠페인 카드로 만드는 메뉴
   - Sponsor Package: 인기 장면과 팬 반응을 스폰서 제안서 형태로 묶는 메뉴
   - Fan Voice: 팬 의견/투표를 수집하고 AI가 분류하는 확장 메뉴
   - Matchday Ops: 직관메이트, 좌석 선호, 경기장 불만, 이벤트 반응을 모아 경기일 운영 개선에 쓰는 메뉴
   - Funnel Detail: 하이라이트 조회부터 예매/굿즈 클릭까지 단계별 전환을 분석하는 메뉴

IR에서 강조할 구현 메시지:

> CourtLens의 구단 대시보드는 지표를 보여주는 화면이 아니라, 팬 반응을 콘텐츠 생성, 캠페인 실행, 스폰서 상품화까지 연결하는 액션형 대시보드다.

## 8. AI Feasibility

현재 프로토타입은 AI 생성 경험을 UI로 시연한다. 실제 제품에서는 다음 파이프라인으로 구현한다.

### MVP Pipeline

1. 구단 또는 리그가 제공한 경기 영상/공식 클립 확보
2. 경기 기록 및 play-by-play 데이터 수집
3. 영상 속 경기 시간 OCR 또는 중계 자막/ASR로 타임라인 동기화
4. 득점, 어시스트, 리바운드, 스틸, 3점 성공 등 이벤트 앞뒤 8~12초 클립 추출
5. 선수별/장면별 태깅
6. 사용자가 선택한 선수 조합에 맞춰 클립 재정렬
7. LLM으로 초보자용 해설과 농잘알용 전술 설명 생성
8. 팬 앱 행동 로그 수집
9. 선수/콘텐츠/직관메이트 데이터를 결합해 구단용 수익화 액션 추천

### Advanced Pipeline

- 선수/등번호 인식
- 공/선수 위치 추적
- 코트 라인 인식
- 픽앤롤, 속공, 아이솔레이션, 미스매치 등 플레이 유형 분류
- 작전판 형태의 시각화 자동 생성
- 구단 CRM과 연동한 팬 세그먼트별 캠페인 추천

### Risk & Response

영상 권리 문제:

- 무단 클립 서비스가 아니라 구단/리그/OTT 제휴형 B2B 플랫폼으로 설계한다.
- 초기에는 구단 제공 영상, 공식 숏폼, 자체 제작 콘텐츠를 활용한다.

전술 분석 정확도:

- 처음부터 영상만으로 모든 전술을 맞히지 않는다.
- 기록 데이터, 경기 시간 OCR, 중계 ASR, 수동 태그를 결합해 MVP 정확도를 확보한다.

개인정보/팬 데이터:

- 개인 식별보다 세그먼트 단위 분석을 우선한다.
- 구단에는 익명화된 팬 반응과 집계 지표 중심으로 제공한다.
- 직관메이트는 공개 범위, 신고, 차단, 안전 가이드를 필수로 둔다.

구단 도입 난도:

- 기존 앱을 대체하지 않고, SDK/웹뷰/관리자 대시보드 형태로 붙일 수 있게 설계한다.

## 9. Business Model

팬에게 직접 과금하는 모델은 핵심이 아니다. CourtLens의 수익 모델은 구단과 스폰서 중심이다. 단, 이 수익 모델은 팬 경험이 충분히 좋다는 전제에서만 작동한다.

구단에게 판매하는 핵심 상품은 `구단 대시보드`다. 이 대시보드는 단순 통계 화면이 아니라 팬 앱에서 발생한 관심, 하이라이트 생성, 작전판AI 조회, 직관메이트 참여, 티켓·굿즈 CTA 반응을 묶어 구단의 실행 전략으로 바꾸는 B2B SaaS다.

구단 대시보드의 상품 구성:

- Fan Engagement Dashboard: 선수별·장면별 팬 반응, 하이라이트 생성률, 저장/공유/클릭 지표
- Content Queue: 반응이 높은 선수와 장면을 기반으로 숏폼, 카드뉴스, 입덕패스 콘텐츠 추천
- Revenue Playbook: 하이라이트 관심을 티켓, 굿즈, 직관메이트, 이벤트로 연결하는 액션 추천
- Sponsor Package Report: 팬이 몰입한 장면과 반응 데이터를 스폰서 제안서로 전환
- Fan Voice Extension: 향후 팬 의견함과 투표를 붙여 경기장 경험, 굿즈, 이벤트 니즈를 분석

즉 CourtLens의 B2B 수익화는 "팬 데이터를 판다"가 아니라 "팬이 더 재밌게 본 결과를 구단의 실행 가능한 매출 전략으로 바꿔준다"에 가깝다.

### 1. 구단 SaaS 사용료

구단이 시즌 단위 또는 월 단위로 대시보드 사용료를 낸다.

제공 가치:

- 선수·콘텐츠별 팬 반응 분석
- 하이라이트 생성/저장/공유 데이터
- 직관메이트 참여 데이터
- 티켓·굿즈·이벤트 캠페인 추천
- 시즌별 팬덤 리포트

### 2. 초기 구축비

구단의 기존 앱, 티켓 예매, 굿즈몰, CRM, SNS 운영 방식에 맞춰 연동하는 비용을 받는다.

제공 가치:

- 구단별 브랜딩 적용
- 티켓/굿즈 링크 연동
- 구단 관리자 계정 설정
- 캠페인 템플릿 세팅

### 3. 성과형 수수료

CourtLens 캠페인을 통해 발생한 티켓, 굿즈, 이벤트 전환에 대해 일부 수수료를 받는다.

예시:

- 하이라이트 시청 후 홈경기 예매
- 선수 콘텐츠 시청 후 유니폼 구매
- 직관메이트 쿠폰 사용
- 멤버십 가입 전환

### 4. 스폰서 패키지 수익 쉐어

구단이 CourtLens 데이터를 기반으로 스폰서 광고 상품을 판매하면 플랫폼이 일부 수익을 공유받는다.

예시:

- 선수 하이라이트 스폰서
- 작전판AI 전술 해설 스폰서
- 직관메이트 모집/응원석 캠페인 스폰서
- 입덕패스 경기 프리뷰 스폰서

핵심 메시지:

> 팬에게는 경기를 더 재밌게 보는 경험을 제공하고, 구단에게는 그 팬 경험을 오래 유지하며 수익화하는 도구를 판매한다.

## 10. Competitive Difference

기존 KBL/스포츠 앱:

- 일정, 결과, 기록, 뉴스 중심
- 팬이 경기의 의미를 이해하도록 돕는 설명이 부족함
- 좋아하는 선수 조합으로 직접 하이라이트를 만들기 어려움
- 직관 동행/관전 포인트를 하이라이트와 연결하지 못함

유튜브/숏폼 하이라이트:

- 편집자 기준의 공통 하이라이트
- 선수별/취향별 개인화 부족
- 구단이 팬 반응을 직접 소유하기 어려움
- 시청 후 티켓/굿즈/스폰서 전환 흐름이 약함
