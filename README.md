# CourtLens

> KBL 팬이 좋아하는 팀과 선수를 고르면 AI가 개인화 하이라이트와 쉬운 전술 해설을 생성하고, 그 관심을 직관메이트와 구단 수익화로 연결하는 농구 팬 경험 플랫폼

CourtLens는 **AI로 스포츠 진입장벽을 낮추면, 팬은 경기를 더 재밌게 보고 구단은 팬을 더 오래 볼 수 있다**는 문제의식에서 출발한 아이디어톤 프로토타입입니다.

팬에게는 좋아하는 선수 중심으로 경기를 쉽게 이해하고 다음 직관까지 이어지는 경험을 제공하고, 구단에게는 팬 반응을 콘텐츠 생성, 이벤트 실행, 스폰서 상품화로 연결하는 액션형 대시보드를 제공합니다.

## 문제 정의

프로농구는 재미없는 스포츠가 아닙니다. 문제는 라이트 팬이 그 재미를 이해하기까지 넘어야 할 장벽이 높다는 점입니다.

- 전체 경기를 볼 시간은 부족하지만, 좋아하는 선수의 장면만 빠르게 모아보기 어렵습니다.
- 픽앤롤, 스페이싱, 미스매치 같은 전술을 모르면 좋은 플레이를 놓치기 쉽습니다.
- 기존 스포츠 앱은 일정, 결과, 기록, 뉴스 중심이라 라이트 팬이 앱을 계속 켤 이유가 약합니다.
- 혼자 직관을 가기 부담스럽고, 같은 선수나 장면을 좋아하는 팬을 찾기 어렵습니다.
- 구단은 팬들이 어떤 선수, 장면, 콘텐츠에 반응하는지 파악하고 티켓·굿즈·스폰서 매출로 연결하기 어렵습니다.

## 솔루션 요약

CourtLens는 팬 앱과 구단 대시보드로 구성됩니다.

팬 앱에서는 사용자가 좋아하는 팀과 선수를 고르면 AI가 개인화 하이라이트와 쉬운 해설을 생성합니다. 생성된 관심은 입덕패스와 직관메이트로 이어져 라이트 팬의 이해와 다음 행동을 돕습니다.

구단 대시보드에서는 팬 앱에서 발생한 선수 선택, 하이라이트 생성, 작전판AI 조회, 직관메이트 참여, 티켓·굿즈 CTA 반응을 분석해 콘텐츠 생성과 수익화 액션으로 연결합니다.

## 주요 기능

### 팬 앱

- **팀/선수 선택**: 로그인된 사용자가 선호팀을 바꾸고, 팀 로스터에서 보고 싶은 선수를 여러 명 선택합니다.
- **AI 하이라이트 생성**: 선택한 선수 조합을 기준으로 AI가 영상을 만든다는 전제로 `방금 생성된 클립`에 새 카드가 추가됩니다.
- **작전판AI**: 하이라이트 장면을 초보 모드와 농잘알 모드로 설명합니다.
- **입덕패스**: 처음 보는 팬도 따라갈 수 있도록 선수와 팀의 관전 포인트를 짧게 제공합니다.
- **직관메이트**: 관심 선수, 경기, 좌석, 응원 성향을 기반으로 같이 직관 갈 사람을 찾는 모집글을 생성합니다.

### 구단 대시보드

- **KBL 10개 구단 데이터 패널**: 전체 구단의 등록 선수, 기록 선수, AI 클립 후보 수를 비교하고 구단별 대시보드로 전환합니다.
- **선수별 팬 반응**: 선수별 하이라이트 조회, 저장, 공유, 직관메이트 클릭을 통합한 반응 점수를 보여줍니다.
- **콘텐츠 자동 생성 큐**: 반응이 높은 선수와 장면을 기반으로 숏폼, 카드뉴스, 입덕패스, 직관메이트 문안을 제안합니다.
- **장면별 수익화 추천**: 인기 장면을 응원석 예매 CTA, 굿즈 이벤트, 스폰서 패키지로 연결합니다.
- **생성 결과함**: 이벤트, 리포트, 스폰서 제안서 등 생성된 문서를 확인하고 다운로드할 수 있습니다.

## 현재 프로토타입 범위

현재 구현된 프로토타입은 실제 KBL 앱과 유사한 기본 흐름을 가지되, CourtLens의 핵심 차별점인 **하이라이트 분석**, **직관메이트**, **액션형 구단 대시보드**를 중심으로 시연합니다.

구현된 범위:

- KBL 공식 API 기반 정적 데이터 사용
- 10개 구단 데이터 패널
- 팀별 컬러, 선수 목록, 주요 기록 리더 자동 반영
- 선수 다중 선택 및 AI 하이라이트 생성 인터랙션
- 생성된 클립 선택 시 작전판AI 설명 갱신
- 직관메이트 모집글 작성 모달 및 카드 생성
- 구단 대시보드 상세 모달, 콘텐츠 미리보기, 이벤트 실행 플로우
- 생성 결과함 문서 미리보기 및 다운로드

아직 구현하지 않은 범위:

- 실제 경기 영상 업로드 및 자동 클립 추출
- 실제 play-by-play와 영상 타임라인 동기화
- 사용자 계정/서버 저장소 기반 선호팀 관리
- 실제 예매/굿즈몰/구단 CRM 연동
- 실제 구단 담당자 승인 및 배포 플로우

## 실행 방법

정적 HTML/CSS/JS 프로토타입입니다. 별도 패키지 설치 없이 로컬 서버로 실행할 수 있습니다.

```bash
python3 -m http.server 5173 --bind 127.0.0.1
```

브라우저에서 아래 주소를 엽니다.

```text
http://127.0.0.1:5173/index.html
```

`file://`로 직접 열어도 대부분 동작하지만, 로컬 서버로 확인하는 방식을 권장합니다.

## 데이터 기준

- `kbl-data.js`는 KBL 공식 API 기반으로 수집된 정적 데이터입니다.
- `scripts/fetch-kbl-data.mjs`는 선수/기록 데이터를 다시 가져오기 위한 스크립트입니다.
- 현재 프로토타입은 브라우저에서 실시간 크롤링하지 않고, 저장된 정적 데이터를 사용합니다.
- AI 하이라이트 생성, 작전판AI 해설, 이벤트/문서 생성은 실제 AI 파이프라인이 붙었다는 전제의 UI 목업입니다.

실제 제품에서는 경기 영상 또는 구단 제공 클립, play-by-play 데이터, OCR/ASR, 영상 클립 추출, LLM 기반 해설 생성, 팬 행동 분석을 결합해 구현합니다.

## 파일 구조

```text
.
├── index.html                  # 팬 앱 + 구단 대시보드 프로토타입
├── styles.css                  # 전체 UI 스타일
├── app.js                      # 인터랙션, 팀 전환, 모달, 생성 결과함 로직
├── kbl-data.js                 # KBL 공식 API 기반 정적 데이터
├── scripts/
│   └── fetch-kbl-data.mjs      # KBL 데이터 수집 스크립트
├── court-lens-ir-draft.md      # IR Deck 및 발표 내용 정리 문서
└── outputs/                    # 발표자료 PDF/PPTX 및 슬라이드 이미지
```

## 기술 스택 및 예상 역할

현재 프로토타입:

- Frontend: HTML, CSS, JavaScript
- Data: KBL 공식 API 기반 정적 JS 데이터
- Prototype: 정적 웹 프로토타입

실제 제품 확장 시 예상 스택:

- Frontend: React 또는 Next.js
- Backend: FastAPI 또는 Node.js
- AI/Video: Python, OpenCV, OCR, ASR, LLM API
- Data: PostgreSQL, Redis, event tracking
- Infra: Vercel, AWS S3, Cloud Run

예상 역할:

- 기획/PM: 문제 정의, 팬/구단 사용자 흐름, B2B 수익 모델, 발표 스토리 구성
- 프론트엔드: 팬 앱 UI, 선수 선택, 하이라이트 생성 인터랙션, 직관메이트, 구단 대시보드 구현
- 백엔드: 사용자/선수/경기 데이터 관리, 선호팀 저장, 팬 행동 로그, 대시보드 API
- AI/데이터: 영상 클립 추출, 기록 데이터 동기화, 선수/장면 태깅, 전술 설명 생성, 수익화 추천
- 디자인: 모바일 앱 화면, 팀별 브랜딩, 생성된 클립 카드, 구단 대시보드 화면, IR Deck 시각화

## 비즈니스 모델

CourtLens는 팬에게 먼저 과금하는 모델보다 구단과 스폰서 중심의 B2B 모델을 우선합니다.

- 구단 SaaS 사용료: 선수별·장면별 팬 반응, 하이라이트 생성률, 저장/공유/클릭 지표 제공
- 초기 구축비: 구단별 브랜딩, 티켓/굿즈 링크, 관리자 계정, 이벤트 템플릿 세팅
- 성과형 수수료: 하이라이트 시청 후 예매, 굿즈 구매, 직관메이트 쿠폰 사용 등 전환 기반
- 스폰서 패키지 수익 쉐어: 인기 장면과 팬 반응 데이터를 활용한 스폰서 제안서 생성
